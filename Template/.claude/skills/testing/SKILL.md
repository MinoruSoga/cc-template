# テストスキル

## このスキルを使用するタイミング

- 新機能のテスト作成
- 既存テストの修正
- テストカバレッジの向上

## テスト種別

| 種別 | コマンド |
|------|---------|
| ユニット | `make test` |
| 統合 | `make test-integration` |
| E2E | `make test-e2e` |

## テスト作成パターン

### ユニットテスト（AAA パターン）

```
describe('関数名', () => {
  it('should [期待される動作]', () => {
    // Arrange
    // Act
    // Assert
  });
});
```

## 重要な注意事項

- モックは最小限に
- テストデータはファクトリパターンを使用
- 非同期テストは適切にawait
- テストフレームワークはプロジェクトの設定に従う
