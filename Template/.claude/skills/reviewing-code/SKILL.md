# Reviewing Code

読み取り専用のコードレビュー。変更は行わない。

## Checklist

### 1. 型安全性
- [ ] `any` の使用を避けているか
- [ ] 適切な型定義があるか
- [ ] nullチェックは十分か

### 2. エラーハンドリング
- [ ] try-catchが適切に使用されているか
- [ ] エラーメッセージはユーザーフレンドリーか
- [ ] ログは適切か

### 3. セキュリティ
- [ ] XSS対策（入力のサニタイズ）
- [ ] SQLインジェクション対策（パラメータ化クエリ）
- [ ] CSRF対策

### 4. パフォーマンス
- [ ] 不要な再レンダリングはないか
- [ ] N+1クエリはないか
- [ ] 適切なキャッシュ戦略か

### 5. 保守性
- [ ] コードは読みやすいか
- [ ] 関数は単一責任か
- [ ] 重複コードはないか

## Output Format

```markdown
## 良い点
- ...

## 改善提案
- ...

## 確認事項
- ...
```
